# 模擬患者資料說明

## 概述

已成功建立 12 位模擬患者，每位患者都有完整且多樣化的健康數據。

## 執行方式

### 步驟 1: 生成資料檔案

```bash
# 方法 1: 使用 npm script
npm run seed-patients

# 方法 2: 直接執行
node scripts/seedPatients.js
```

這會在 `data/` 目錄下建立 JSON 資料檔案。

### 步驟 2: 匯入資料到應用程式

有兩種方式可以將資料匯入到應用程式：

#### 方法 A: 使用網頁導入工具（推薦）

1. 啟動開發伺服器：
   ```bash
   npm run dev
   ```

2. 開啟瀏覽器前往：
   ```
   http://localhost:8080/import-data.html
   ```

3. 點擊「開始匯入資料」按鈕

4. 匯入完成後，返回主應用程式即可看到患者資料

#### 方法 B: 手動複製到 localStorage

如果方法 A 無法使用，可以：

1. 啟動應用程式
2. 開啟瀏覽器開發者工具 (F12)
3. 在 Console 中執行以下指令來手動導入資料

## 資料特色

### 1. 體組成記錄 (Body Composition)
每位患者有 8-15 筆歷史記錄，分散在過去 180 天內，包含：
- 體重 (kg)
- 身高 (cm)
- 體脂率 (%)
- 肌肉量 (kg)
- BMI
- 內臟脂肪等級
- 骨量 (kg)
- 體水分 (%)
- 基礎代謝率 (kcal)

### 2. 生命徵象記錄 (Vital Signs)
每位患者有 6-12 筆歷史記錄，分散在過去 120 天內，包含：
- 收縮壓 / 舒張壓 (mmHg)
- 心率 (bpm)
- 體溫 (°C)
- 呼吸率 (次/分)
- 血氧飽和度 (%)
- 血糖 (mg/dL)

**特殊設計**：
- 有「高血壓」標籤的患者，血壓數值較高（130-160 / 85-100）
- 有「糖尿病」標籤的患者，血糖數值較高（120-180）

### 3. 預約記錄 (Appointments)
每位患者有 4-7 筆預約，包含：
- 過去已完成的預約
- 未來待完成的預約
- 多種預約類型：定期檢查、營養諮詢、運動指導、複診、健康評估

### 4. 健康目標 (Patient Goals)
根據患者標籤自動生成相應目標：
- **體重目標**：給減重計畫的患者
- **體脂率目標**：給減重或健身的患者
- **運動目標**：給運動員或健身的患者

每個目標包含：
- 起始值和目標值
- 開始日期和目標日期
- 當前進度 (0-100%)
- 狀態追蹤

### 5. 初始評估 (Initial Assessment)
每位患者都有完整的初始健康評估，包含：
- 基線測量值（體重、身高、體脂、肌肉量、BMI）
- 健康目標設定
- 活動水平評估
- 評估日期和評估人員

## 使用說明

1. **執行腳本**：執行 `npm run seed-patients` 建立模擬資料
2. **重新載入應用程式**：關閉並重新開啟應用程式
3. **查看患者**：在患者列表中會看到新增的 12 位患者
4. **查看詳細資料**：點擊任一患者可查看完整的健康記錄、圖表和目標進度

## 重複執行說明

- 腳本會檢查患者電話號碼，避免重複新增
- 如果患者已存在，會自動跳過該患者
- 可以安全地重複執行腳本

## 資料真實性

所有模擬資料均符合醫學常理：
- 正常體重範圍：BMI 18.5-30
- 正常血壓範圍：110-160 / 70-100 mmHg
- 正常心率範圍：60-90 bpm
- 正常體溫範圍：36.2-37.2°C
- 正常血氧飽和度：95-100%
- 體重變化趨勢模擬漸進式改變
